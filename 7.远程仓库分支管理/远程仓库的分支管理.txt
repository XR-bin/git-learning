1、查看远端仓库分支情况
- 我们知道查看本地仓库的分支情况可以用  git branch
- 但这个命令无法查看到远端仓库的分支情况
- 这个时候我们可以用下面的命令
    - git branch -a 



2、将本地分支上传到远端仓库分支
- 我们之前 git push 的时候一直都是上传主分支到远端仓库
- 这是因为我们 git push 每次只上传一个分支
- 这个时候如果我们要上传另一个分支到远端仓库，就应该这样写

    - git push origin 本地分支名:远端分支名

    - 注意：远端分支名在远端仓库不存在会自己自动新建
    - 我们可以直接省略:远端分支名
        - git push origin 本地分支名

- 例子：我们有个login分支，要上传到远端仓库
    - git push origin login

    - 注意：上传login分支时，必须切换到login分支上



3、拉拽远端仓库的分支
- 和上传一样，命令如下
    - git pull origin 远端分支名



4、删除远端仓库的某个分支
- 当某个远端仓库的分支我们不需要了，那么我们就可以用下面的命令将远端分支删除
    - git push origin :远端分支名

- 例如我们删除login远端仓库分支
    - git push origin :login
